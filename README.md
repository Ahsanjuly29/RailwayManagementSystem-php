# 🚂 RailFlow: Procedural PHP Railway Management System

![PHP](https://img.shields.io/badge/PHP-777BB4?style=for-the-badge&logo=php&logoColor=white)
![MySQL](https://img.shields.io/badge/MySQL-4479A1?style=for-the-badge&logo=mysql&logoColor=white)

RailFlow is a comprehensive ticket reservation and management platform developed during my undergraduate studies. Built using **Procedural (Raw) PHP**, this project focuses on the core logic of inventory management, user bookings, and real-time availability searching without the use of high-level frameworks or OOP.

## 🌟 Key Functional Highlights
*   **Ticket Reservation Engine**: A complete workflow for users to search, book, and view railway tickets.
*   **Booking Management**: Users can track their booking history (`my_ticket.php`) and perform cancellations (`cancel_booking.php`) with real-time database updates.
*   **Search & Availability**: Dynamic search functionality allowing users to find available trains based on specific criteria.
*   **Admin Control Panel**: Specialized interface for managing train schedules, routes, and user data.
*   **Session-Based Security**: Secure user and admin authentication systems to protect transactional data.

## 🛠️ Technical Implementation
*   **Architecture**: 100% Procedural PHP, focusing on server-side logic and procedural data handling.
*   **Database Design**: Relational MySQL schema optimized for handling seat availability and user reservations.
*   **Frontend**: Responsive and accessible UI built using HTML5, CSS3, and JavaScript.
